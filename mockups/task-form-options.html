이번 주 업무 입력 폼 — UX 비교

두 안을 클릭하며 시각적으로 비교해보세요. 실제 동작은 단순화되어 있습니다.

안 1 — 메인 폼에 KPI / 이전 업무 둘 다

부모 관계(KPI / 이전 task)를 메인 폼에 명시 노출. 모든 case가 분명함. 폼이 길어지는 trade-off.
▶ 어디에 연결? (둘 중 하나는 권장)
▶ 세부 설명
▶ 메타
장점: 모든 case가 명시적. 흐름도 구조와 직접 1:1 대응.
단점: 폼이 길고 입력 부담 ↑. 첫 사용자에 혼란 가능.
입력 case별 흐름도 결과:
· KPI만 → 차트에 KPI → 이 task
· 이전 task만 → 차트에 prev → this (KPI는 prev에서 자동 inherit)
· 둘 다 → 차트에 KPI ↔ prev ↔ this 그래프
· 둘 다 비움 → 독립 task (이런 케이스는 임시/급한 일)

안 3 — 현재 폼 + 후속 자동 추천 팝업

현재 폼 거의 그대로. KPI 선택 시 같은 KPI를 가진 최근 task가 있으면 "이거 후속인가요?" 팝업이 떠서 한 번 클릭으로 부모 연결.
💡 이 KPI에 진행 중 업무가 있어요. 새 업무를 어떻게 등록할까요?
잘뜨는거맞네 3일 전 · 진행 중 · 같은 KPI
홈쇼핑 매출체크 어제 · 진행 중 · 같은 KPI
장점: 폼이 짧고 직관적. 신규 업무 작성이 빠름. 부모 연결은 KPI 선택만으로 자동 제안.
단점: 팝업 추천 인식률에 의존 (아래 분석 참조).
인식률 결정 요소:
· 사용자가 KPI를 잘 채워둘 때만 동작
· 동일 KPI 가진 진행 중 task가 1개 이상 존재해야 함
· "이전 업무 후속" case의 80%는 KPI 같음 → 인식률 70~80% 추정
· 나머지 20%(KPI 다른 후속)는 사용자가 + 업무 연결로 직접 추가
